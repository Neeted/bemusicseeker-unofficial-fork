## 概要

本プロジェクトは、`Sayaka / 黒皇帝` 氏の [BeMusicSeeker-installer](https://github.com/SayakaIsBaka/BeMusicSeeker-installer) に同梱されている `BeMusicSeeker.exe` を `ILSpy` でデコンパイルし、改修・再構築を行った非公式版です。

オリジナルバイナリの作者である [`@rib_2_bit`](https://x.com/rib_2_bit) 氏から直接の許諾を得たうえでのフォークではありませんが、元バイナリが MIT License の下で公開されていることに基づき、派生版の作成および公開は問題ないと判断しています。

**【重要】本リポジトリについて**
オリジナルのソースコードは非公開のままだったと認識しており、本プロジェクトにおいても、デコンパイルによって得られたソースコードそのものは（大部分を）リポジトリに含めていません。
本リポジトリは事実上、**「改修済みリリースパッケージ（バイナリ）の配布」** と **「新規に追加・作成した一部のスクリプトや差分コードの公開」** を目的としたリポジトリとして運用されます。

## 導入手順

1. **[Everything 1.5 Alpha (x64)](https://www.voidtools.com/everything-1.5a/) の導入**
    - ※ 未導入環境でも動作しますが、起動高速化の恩恵は十分に得られません。導入後は、BMS 関連ファイルが存在するディレクトリのデータベース構築が完了し、検索できる状態になっていることを確認してください。
2. **リリース版のダウンロード**
    - [Release](https://github.com/Neeted/bemusicseeker-unofficial-fork/releases) ページから最新版をダウンロードし、任意のディレクトリへ展開・配置してください。インストール作業は不要なポータブルアプリとして動作します。
3. **設定の自動引き継ぎ**
    - 初回起動時、既に従来の BeMusicSeeker がインストールされている環境であれば、設定を自動的に引き継ぎます。
    - 引き継がれた設定は配置先の `.\config\user.config` に保存されるため、元の BeMusicSeeker 本体の動作には影響を及ぼしません。
4. **使い方と新機能について**
    - 基本的な使い方の説明は割愛します。新機能に関する詳細なドキュメントもまだ十分には整備できていないため、ひとまず以下の「変更概要」を参照してください。
    - `LaunchWithInfoLog.bat` から起動すると、通常よりも詳細な `[INFO]` レベルのログが `application.log` に出力されます。動作の調査やトラブルシューティングに役立ちます。ログの見方については [ログレベル INFO ガイド](docs/log-level-info-guide.md) を参照してください。

## 変更概要

主に従来版との差異を紹介します。  
**※ 最も懸念しているのは、BMS 関連のデータベースやファイルが破損することです。導入・利用は自己責任でお願いします。**

### 性能

- **初期化高速化**
  - `Everything 1.5a` 必須（元のファイル列挙もかなり高速でしたが、1000 万ファイル級の環境に対応するため Everything による検索を採用しています）
  - ファイル列挙以外の初期化処理にも手を入れています
- **推定先への移動高速化**
  - 処理遅延の主要因だった移動ごとのUI更新を、バッチ処理寄りに調整しました
- **導入先推定高速化**
  - ロジック自体は旧版から踏襲しつつ、ハッシュ値などを用いて探索速度を強化
- **マージ機能（重複ファイルチェック）の高速化**
  - 重複ファイルチェックおよびマージ処理のロジックを最適化し、大幅な高速化を実現しました。
- **全体的にUIの負荷軽減**
  - 画面更新タイミングの見直しや、画面外の要素を描画しない仮想化を多く取り入れました。
  - 画面の状態と内部データの不整合が発生しやすいというリスクはありますが、随時安定化しつつ、現時点では性能を優先する方針を取っています。
- **一覧画面で高速ソートを使う設定の追加（デフォルト無効）**
  - 「高速ソート」は、従来よりも表示・再ソートの軽さを優先したソート方式です。従来はアプリ内のソートにすべて自然順が適用されていましたが、難易度表のフォルダ名は自然順のまま維持しつつ、一覧表示に限って高速ソートを適用できるようにしました。一覧表示での `PATH` や `TITLE` が単純な文字列順でも問題ない場合は、より高速に表示できます。
- **プレイリスト明細表示の高速化**
  - プレイリスト表示用データの持ち方を見直し、表示専用の軽量な行データへ整理しました。
  - 10万譜面規模のプレイリストでも、初回表示・切り替え・再ソート時の負荷を抑え、よりスムーズに表示できるようにしています。

### 機能別

#### インストール

- **同梱追加譜面の差分導入対応**
  - 従来は「既所持譜面を含むフォルダ」の場合、未所持の追加譜面のインストール先が推定されなかった問題を修正しました。
- **スマート上書きモード追加（デフォルト有効）**
  - 同梱されている音源や BGA などを導入する際、新しいファイルを古いファイルで上書きしてしまわないように、更新日時を比較して確認するように変更しました。
  - スマート上書きの拡張設定として `スマート上書き時、*.bmx/*.pmx/*.txt/*.bmson は上書きせず自動採番で保持する` （デフォルト無効）も追加しています。同梱の `readme.txt` を差分側の `readme.txt` で上書きしてしまう、といった事故を防ぐ意図です。
- **アーカイブ（zip等）導入時のファイル更新日時復元**
  - アーカイブをドラッグ＆ドロップして導入する際、展開されたファイルの更新日時が元のアーカイブ内のものと一致するように修正しました。これにより「スマート上書きモード」がより正確に機能するようになります。
- **保留画面での実ファイル削除対応**
  - インストール済みの差分実ファイルを、保留画面上からも直接削除できるようにしました。
  - 保留画面での `ごみ箱へ移動` には `[x] 対象フォルダにBMSファイルが含まれなくなる場合フォルダごと削除する。` という機能が実装してあります。
  - ライブラリ画面などでの削除に近い挙動です。保留画面で `ごみ箱へ移動` を選択した際の確認ダイアログにこのチェックボックスがあります（デフォルト有効）。有効のまま OK を押すと、選択範囲内のすべての BMS パッケージにこの処理が適用されます。ライブラリ画面などでの削除と違い、フォルダごとの確認ダイアログは表示されません。
- **差分導入先の手動入力許可**
  - 推定に失敗した場合の救済措置として、既所持 BMS フォルダのパスを手動で入力できるようにしました。
  - ※たまにしか使用しない想定のため、「第二候補のサジェスト」のような便利機能は省いています。また、セルを編集しようとクリックした際に再生が始まってしまう既知の仕様があります。
- **「インストール先を開く」機能を追加**
  - 保留画面のツリーや明細の右クリックメニューから、推定先や既所持ファイルの導入先フォルダを直接開けるようにしました。
- **保留画面における「マージ先を推定」機能を追加**
  - 既所持の BMS 譜面に対し、リソース（音源や BGA など）の修正パッチを適用しやすくするための機能です。
  - インストール済みの譜面であっても意図的に導入先を再推定させることで、譜面自体に変更がない場合でもリソースのみを上書き更新することが可能になります。
- **新規/保留 で選択範囲のBMSパッケージを `一覧から削除` する右クリックメニューを追加**
  - ツリーで右クリックした際に個別に `一覧から削除` することは従来からできていましたが、これを明細から複数選択した場合にも呼び出せるようにしました。
- **「通常インストール時、既所持譜面が残っていても元パッケージを削除する」設定追加(デフォルト無効)**
  - 従来は通常インストール時は `既にインストールされているファイルはスキップされます。` とダイアログに書いてある通り、既所持譜面は元フォルダに残される仕様でした。これを重複ファイルチェックのマージ処理に近い使用感にするための設定を追加しました。有効にするとパッケージ内の新規譜面を導入した後の元フォルダを残り物ごと削除する挙動になります。
- **保留ツリーに「既所持譜面のみのパッケージを完全削除」機能を追加**
  - 保留ツリーの右クリックに「高度な機能>既所持譜面のみのパッケージをごみ箱を経由せずに削除」を追加しました。既所持と未所持が混在し混沌としたときに使う想定です。
- **保留ツリーに「既所持譜面のみのパッケージをリソースのみ上書きする」機能を追加**
  - 保留ツリーの右クリックに「高度な機能>既所持譜面のみのパッケージをリソースのみ上書きする」を追加しました。音源やBGAに更新があった場合に使う想定です。
- **保留ツリーに「ゼロノート譜面を無効な拡張子に変更(*.bmx/pmx)」機能を追加**
  - 保留ツリーの右クリックに「高度な機能>ゼロノート譜面を無効な拡張子に変更(*.bmx/pmx)」を追加しました。未所持譜面かと思ったら `.bmx` 化されていた既所持ファイル、ということがよくあるので、その対策として導入しました。
- **複数パッケージをドラッグ&ドロップ導入した際の挙動を改善**
  - 従来は受け取ったパスを 1 つずつ同期的に推定完了まで処理していたため、ドロップ元のロック時間が長くなっていました。先にパスだけまとめて受け取り、その後に処理することでロック時間を最小化しています。

#### プレイリスト

- **難易度表リロード時のタイムアウト延長（30秒 → 300秒）**
  - スプレッドシート等で管理されている巨大な難易度表では、レスポンスに 30 秒以上かかるケースがあったためです。
- **推定難易度表/リコメンド表（自動更新含む）の復活**
  - 難易度表情報の取得先・依存先を [DARKSABUN](https://darksabun.club/) へ変更（※更新されない前提であればローカル保持でも良かったかも）
- **プレイリストサマリーの追加**
  - 未所持譜面が増えた際にすぐ気が付けるようにしました。
  - 完全所持 / 未所持ありでの絞り込み機能があります。右上の検索ボックスから設定できます。
  - プレイリストサマリー行をダブルクリックで対象のプレイリスト画面へ自動遷移する機能があります。サマリー画面で未所持が増えていた場合に、プレイリストへ遷移し `[NO SONG]` を確認するような用途を想定しています。導入している難易度表が多い場合に便利だと思います。
  - `STATUS` カラムを追加しました。アプリ起動後に行われた外部プレイリスト同期の最新結果を表示し、リンク切れやヘッダ/データ取得失敗といったエラーに気が付きやすくしています。詳細は tooltip で確認できます。
- **プレイリストの個別リロード機能を追加**
  - プレイリストツリーやプレイリストサマリー明細の右クリックメニューから表の単体リロードができるようにしました

#### 重複ファイルチェック

- **重複ファイルチェック画面の微修正**
  - 3つ以上の重複が存在する場合に警告文が複数行表示されていた問題を修正し、重複警告を1件に集約しました。
  - 重複している譜面の行は、背景色を薄い赤色でハイライトするように変更しました。
  - **※スマート上書きモードの仕様はここにも影響しています。**
- **重複フォルダマージ(Ctrl+G)の機能拡張**
  - **概要**: 重複ファイルチェックツリーにおいて、**統合したい（消したい）方のフォルダを選択した状態**で `Ctrl + G` を押すことで、マージ作業をショートカットできます。
  - **マージ動作の自動切り替え**:
    - 重複が2件（1対1）の場合は、統合先が一意に定まるためそのままマージを実行します。
    - 3件以上のファイルが絡む場合は、統合先を選択できるようコンテキストメニューのリストを自動的に展開します。
  - **単一フォルダ内の重複解消**: ハッシュのみ重複している単一フォルダ内の重複解消ができるようになりました。
  - **連続処理の効率化**: マージ実行後、ツリー内の「次に処理すべき場所」へ自動的にフォーカスが移動するようになりました。リストの上から `Ctrl + G` を連打するだけで、大量の重複をスピーディーに整理できます。

#### ゼロノート検索

- **`ゼロノート再判定` 機能を追加**
  - `ゼロノート検索` ツリーの右クリックメニューに追加しました。LR2 または BeMusicSeeker が song テーブルに `karinotes = 0` を記録している譜面のうち、実ファイル上は可視ノートがある「怪しい譜面」を手動で洗い出せるようにしました。WARNING列に `DBに0notesと記録されていますが可視ノートが存在します` と表示されます。重複ファイルチェックと同様にこの判定に引っかかった譜面は行が着色されます。LR2ではパースエラーなどの異常で再生できないが、他のプレイヤーや譜面ビューワー上は再生できる譜面が該当する可能性があります。

#### 文字化けチェック

- **手動での文字化け修正先に `簡体字中国語(GB2312)`, `繁体字中国語(Big5)` を追加**
  - 手動修正の候補に `簡体字中国語(GB2312)` と `繁体字中国語(Big5)` を追加しました。

### その他

- **左サイドバーの上下二段化**
  - 大量の難易度表を登録している環境でのUI操作性を向上
- **ステータスバーの活用範囲を拡張**
  - ステータスバーを、初期化・ライブラリリロード進捗、外部プレイリスト同期進捗、導入先推定進捗の表示に利用するようにしました。
- **ネットワーク関連の刷新**
  - **HTTPリクエストでのUser-Agentヘッダ付与**: 一部の難易度表サイトにおいて発生していた `403 Forbidden` の問題に対処しました。UAが付与されていないアクセスは拒否される場合がありました。
  - **`WebClient` から `HttpClient` への移行**: 非推奨となっている `WebClient` の利用箇所を `HttpClient` へ置き換えました。性能への寄与は少ないと思われますし、デグレのリスクもありますが将来性を考えての移行判断でした。
- **ポータブルアプリ化**
  - 初回起動時、既存のBeMusicSeekerがシステムにインストール済みの場合は、その設定ファイルを自動的に引き継ぐようにしました
- **アプリの更新チェック機能（簡易版）**
  - 起動時、リポジトリに新しいバージョンが存在する場合にダイアログを表示してお知らせします。
  - ※簡易的な通知機能のため、自動アップデート機能はありません。該当のダイアログが表示された場合は、手動で最新版のダウンロードと上書き更新をお願いします。
  - 従来版との差異というよりは本Fork版にも更新チェック機能がありますというお知らせです。
- **多言語対応の強化**
  - 英語、フランス語の対応強化に加え、新たに中国語（簡体字）、中国語（繁体字）、韓国語に対応しました。
- **変更系ファイル操作の実装を抜本的に見直し（上書き / 移動 / 名前変更 / 削除 / 更新日時変更など）**
  - 従来版のファイル操作は `System.IO.File` クラスを直接使っている箇所が多く、ファイル属性によるアクセス拒否が発生しやすい状態でした。本 Fork ではファイル操作用の共通サービスを用意し、これを経由して処理することで、BMS インストールに関わる操作全般でアクセス拒否による中断が発生しにくくなりました。権限のイメージとしては `Windows Explorer` によるファイル操作に近く、読み取り専用属性が付いていても削除や上書きができる点も同様です。BMS ファイル操作に直接関わらない部分は従来仕様のままです。
- **`無効な拡張子に変更(*.bmx/pmx)` で名前衝突時に自動解決**
  - 従来版では名前衝突時に処理が中断され、明細からは消えるが実ファイルは残るという状態でした。
  - 本Fork版では、同一ハッシュ時はリネーム元ファイルを削除し、異なるハッシュ時は自動採番でリネームを行うようにしました。名前衝突のダイアログは出さずに暗黙的に処理が行われます。
- **メモリ使用量とライフサイクル管理の見直し**
  - ライブラリリロード時やプレイリスト切り替え時の参照保持・ライフサイクルを見直し、メモリリーク的に見えていた無制限な積み上がりを抑制しました。
  - 特にプレイリスト切り替え時は、古い表示用データや不要な再構築が残りにくいように整理しています。

    - 具体的な改善案があるわけではないが、重要なコア機能なので動作仕様を再確認しておく必要がある。
12. LR2の設定ファイルやDBを同梱する(song.db, config.xml)
    - 設定ファイルとDBさえあればLR2本体は動作に必須ではないので同梱してしまえばLR2本体の入手が必須要件ではなくなる。
    - 特に工夫しないとLR2本体がないのに意味もなく.lr2folderが作られる状態になると思われるがそこはご愛敬ということにする。
    - beatorajaへの完全対応はBeMusicSeekerをこねくり回してやるほどの利点はないと思われるので行わない。DB更新に限れば[十分に高速な方法](https://github.com/Neeted/songdata-updater)が既にある。
13. 導入先推定アルゴリズムの改善
    - 対症療法として `INSTL DST` の手動入力を許可したが、定義が少なかったり、連番だったりの誤判定対策としてタイトルなども考慮に入れたい。
    - 方法は全く考えていないけど、これを行った後は導入ミス疑いの検知もできそう。
14. `.wav` と `.ogg` が重複して存在している場合にどちらかに寄せる機能
